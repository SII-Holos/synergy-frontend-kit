# Product

## Register

product

## Users

Synergy users who install official plugins, frontend-focused agents that need reliable design skills and MCP tools, and maintainers who publish signed plugin releases. They are usually inside a project workflow and need quick confidence that the plugin is installed, current, and safe to use.

## Product Purpose

Synergy Frontend Kit gives Synergy agents an official frontend capability layer: curated design skills, shadcn/ui access, layout validation, visual verification, and repeatable upstream skill synchronization. Success means a user can install the plugin, understand its permissions, use the skills immediately, and trust that maintainers can refresh upstream bundles without manual drift.

## Brand Personality

Precise, composed, expert. The plugin should feel like an official instrument panel for frontend craft, not a marketing page or decorative showcase.

## Anti-references

Avoid generic AI SaaS styling, purple gradient hero language, card-heavy marketing layouts, decorative glassmorphism, and noisy dashboard chrome. Do not hide shell/network risk behind vague copy.

## Design Principles

- Make trust visible: show source, version, verification, permissions, and release status plainly.
- Keep workflows direct: setup, sync, verify, and publish should each have an obvious path.
- Prefer quiet density: the UI should be scannable and useful in a narrow panel.
- Practice the plugin's standards: accessible text, restrained motion, clear hierarchy, and no decorative excess.
- Preserve upstream provenance: bundled skills must remain traceable to their source repositories.

## Accessibility & Inclusion

Target WCAG 2.1 AA for plugin UI. Respect reduced motion, keep text contrast high, avoid color-only status communication, and make all controls keyboard reachable.
